You are working on Evident (React/TypeScript web app). Implement “Vertical Landing Pages” and an in-app “Mode” system so users can enter the same core workspace (Upload → Ask → Get answers) but with different default tools, shortcuts, and copy depending on the selected category.

GOALS
1) Create separate landing pages (routes) for:
   - /students
   - /educators
   - /finance
   - /legal
   - /hr
   Each page should have a tailored hero section and CTA that routes into the main app/workspace while preserving a selected “mode”.

2) Implement a Mode system in the main app:
   - Modes: "general" | "students" | "educators" | "finance" | "legal" | "hr"
   - The UI stays the same, but:
     a) default selected tab/tool changes
     b) suggested “quick action” buttons change
     c) optional banners/tooltips change
     d) irrelevant student promo banners should not show in non-student modes

3) Mode selection should persist:
   - If user arrives from landing page, set mode from URL param or route
   - Save to localStorage so future visits keep the last mode
   - If user is logged in and you already store user preferences, also store mode in the user profile (optional, only if easy)

4) Add optional URL parameter support:
   - Support ?mode=students etc on any route.
   - Priority order for determining mode:
     (1) explicit ?mode=
     (2) landing route (/students sets students, etc)
     (3) localStorage
     (4) default "general"

5) Inside the main workspace, implement:
   - a “Mode Switcher” dropdown in the top bar (simple)
   - When changed, it updates localStorage and updates UI immediately.

UI REQUIREMENTS (Minimal changes, maximum impact)
