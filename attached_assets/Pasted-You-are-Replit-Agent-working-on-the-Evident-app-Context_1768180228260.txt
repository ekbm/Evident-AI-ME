You are Replit Agent working on the Evident app.

Context
- Frontend: React + TypeScript + Vite + Tailwind + shadcn/ui + Wouter + TanStack Query
- Backend: Node.js + Express + TypeScript + PostgreSQL + Drizzle ORM
- Finance module exists (Invoice Reconciliation) with document extraction (PyMuPDF service) producing `extracted_json`.
- Reconciliation engine produces `reconciliations` + `reconciliation_findings`.

Goal
Add a clean “Manual Correction & Review” workflow that lets users:
1) Correct extracted invoice/PO/GRN data BEFORE reconciliation (normalization corrections)
2) Accept / Override reconciliation findings AFTER reconciliation (human-in-the-loop decisions)
All changes must be fully auditable and source-backed, without turning Evident into an accounting system.
No payments. No ERP record editing. Only “normalized snapshot + review decisions + export”.

Hard requirements
- Strict audit trail: who changed what, when, and why.
- Never overwrite original extraction. Store versions/snapshots.
- Manual corrections must be explicit and visible (banner + change history).
- Override requires a mandatory note.
- Export is allowed only when all CRITICAL findings are resolved (accepted/overridden) OR fixed via normalization + rerun.

1) DB schema updates (Drizzle migrations)
2) Backend API routes for corrections + reviews
3) Diff/patch utilities (pure TS)
4) Frontend UI for:
   - “Review Extraction” corrections

   - “Change History / Audit” viewer
5) Tests for key logic + API
6) Update docs/README

== BACKEND API ==
Create/extend routes under `/api/finance`:

1) Normalization (pre-reconciliation)
- PUT `/api/finance/documents/:id/normalize`
  body:
   {
     "normalized_json": { ... },     // full normalized doc JSON (invoice/po/grn)
     "note": "Fixed total and line item qty"
   }
  behavior:
   - validate user auth + org scope
   - ensure `extracted_json_original` is set (if missing, set it from existing `extracted_json`)
   - set `extracted_json_normalized` = normalized_json
   - set `active_extracted_json` = normalized_json
   - set status = normalized, set normalized_by/at
   - write row to `finance_document_changes` with patch_json computed from original->normalized
   - return updated finance_document + computed diff summary

- POST `/api/finance/documents/:id/reset-normalization`
  behavior:
   - set `extracted_json_normalized` null
   - set `active_extracted_json` = extracted_json_original
   - status = raw
   - write audit change row (change_type normalize_reset)

2) Findings review (post-reconciliation)
- POST `/api/finance/reconciliations/:id/findings/:findingId/review`
  body:
   { "decision": "accepted" | "overridden", "note": "..." }
  rules:
   - note REQUIRED for overridden
   - store in reconciliation_reviews
   - return updated finding resolution (latestReview + resolved)

3) Mark reconciliation reviewed (overall)
- POST `/api/finance/reconciliations/:id/mark-reviewed`
  body: { "note": "Reviewed all variances; approved for export" }
  behavior:
   - only allowed if all critical findings are resolved OR there are no critical findings
   - store reconciliation_reviews with decision=mark_reviewed
   - update `reconciliations.status` -> `reviewed`
   - return updated reconciliation

4) Get audit history
- GET `/api/finance/documents/:id/history`
  returns:
   - original/normalized metadata
   - list of finance_document_changes with diff summary

- GET `/api/finance/reconciliations/:id/audit`
  returns:
   - reconciliation_reviews timeline
   - per-finding latest decision

5) Export gating enforcement
- In export endpoints, enforce:
  - if any CRITICAL findings unresolved => 409 with friendly message
  - else allow export
