You are a senior product engineer + UX designer building “Evident Live” inside our existing Evident web app.

GOAL
Add a new public-facing flow on the website that lets users run a lightweight “AI Readiness Assessment” (Evident Live) and, if they want an Agent, they can submit a “Contact Us for Agent” request (no agent download required in v1).

IMPORTANT CONSTRAINTS
- This is a WEB-ONLY v1. No desktop agent build/download in this task.
- The assessment must be SAFE + LIGHTWEIGHT:
  - Prefer folder picker via File System Access API (Chrome/Edge) if possible.
  - If unsupported, fallback to uploading a small sample set of files (multi-file upload).
  - Do NOT attempt deep content parsing for every file in v1; only do basic signals:
    file type, size, lastModified, filename patterns, folder depth, duplicates by name/size, obvious scan PDFs (if detectable), and “metadata presence” heuristics (based on filename patterns).
- Provide a simple score (0–100) with category breakdown and clear recommendations.
- Make the UX friendly and transparent: “We don’t upload your file contents for this scan” (unless user uploads; then clearly say uploads happen).
- After results, provide a CTA: “Need the Agent? Contact us” → opens a form and sends request to backend (or stores in DB).
- Ensure this works on mobile and desktop.
- Keep Apple-review-safe language (no “enterprise surveillance”). Use “opt-in” language.

TECH STACK (match our app)
- React + TypeScript
- Tailwind + shadcn/ui
- Wouter routing
- TanStack Query if needed
- Node/Express backend (existing)
- PostgreSQL (existing) with Drizzle ORM if available

3) Backend
- Add endpoint POST /api/agent-leads
- Validate inputs, store to DB table agent_leads
- Include timestamp + source = “evident_live”
- Return success JSON
- (Optional) send an email notification if we already have email service; otherwise just store lead

4) Scoring logic (v1 heuristic)
Compute a score 0–100 from these signals (document this in code comments):
- Format Compatibility (0–25):
  - modern formats score higher: pdf/docx/xlsx/pptx/txt/md
  - images/scanned PDFs reduce score
- Structure & Naming (0–20):
  - filenames containing dates, versions, owners, project codes increase score
  - “final_final_v7” patterns reduce score
- Duplicates & Version Clarity (0–20):
  - detect duplicates by same name + same size
  - high duplicate ratio lowers score
- Metadata Signals (0–20):
  - if lastModified exists, file sizes reasonable, and naming suggests owner/date
- Searchability Signals (0–15):
  - if many files are images or “scan” patterns, reduce
  - if many are text-friendly formats, increase

  - if not: show upload fallback
- Ensure graceful errors and user-friendly messages.
