Build a full-stack web app MVP called “Evident” in a single Replit workspace.

GOAL
Evident is an “any-file” evidence-based assistant.
A user uploads ONE file (document, image, audio, video, or other), then:
1) Can ask questions about it.
2) App answers using RAG with citations (page/timestamp/segment/chunk references).
3) App can perform one “tool action”: extract an obligations checklist as structured JSON (works best for contracts/policies, but should run for any file and return [] if none).

IMPORTANT CONSTRAINTS (MVP)
- Use OpenAI API (cloud AI). Do NOT use local LLMs.
- Must work end-to-end inside Replit.
- Store data locally in SQLite (no external DB dependency).
- Keep datasets tiny. This is a demo: support one uploaded file at a time, but design DB so multiple assets can exist.
- Use a small local “vector store” by saving embeddings as JSON arrays in SQLite and doing cosine similarity in Node (OK for small chunks).
- Add conservative file limits for Replit stability:
  - Max upload size: 25 MB
  - For video/audio: recommend short clips (<= 2–3 minutes) for faster transcription
- Provide seeded sample files for testing (at least: TXT contract, PNG image with text, short WAV/MP3 audio). If adding a video sample is too heavy, document how to test video with a small user clip.

  OR React + Vite (only if you’re confident it’ll run smoothly in Replit)

Create tables:

- created_at TEXT

artifacts:

chunks:

REQUIRED BEHAVIOR: “ANY FILE” INGESTION
Your app must ACCEPT any file upload.
It should TRY to produce “evidence text” depending on type:

A) Documents
- Supported: PDF, TXT, DOCX (optional: PPTX later)
- PDF: extract text via pdf-parse; citations should attempt page-based if feasible; if not feasible, cite "chunk=N".
- TXT: use file text directly; citations can be chunk-based or line ranges.
- DOCX: extract text via mammoth; citations chunk-based.

B) Images (PNG/JPG/JPEG/WebP)
- Create evidence text using OpenAI vision-capable Responses call:
  - Extract any visible text (OCR-style) + brief description.
- Save as artifact(s): "ocr_text" and/or "image_caption".
- Citations: "image=1" plus chunk index (e.g., "image=1:chunk=2").

C) Audio (MP3/WAV/M4A) and Video (MP4/MOV)
- Produce transcript text using OpenAI transcription endpoint (Audio API).
- Chunk transcript by time windows if timestamps are available; otherwise chunk sequentially and cite "t=unknown:chunk=N".
- Citations: timestamps if you can. If not, cite chunk ids (e.g., "transcript:chunk=4").
NOTE: For MVP, you may implement transcript without timestamps if the SDK response only returns plain text. If timestamps are available, store them.

D) Unknown/Other file types (e.g., CSV, JSON, ZIP)
- If it’s plain-text readable (CSV/JSON), treat as text and chunk.
- If truly unsupported (e.g., binary), set status UNSUPPORTED and store a fallback_note artifact that explains: “Unsupported type; please upload PDF/TXT/DOCX/PNG/JPG/MP3/MP4”.

CHUNKING + EMBEDDINGS
- Chunk size ~1200 chars, overlap ~150.
- Create chunks with a meaningful source_ref:
  - Documents: "page=N" when possible; else "doc:chunk=N"
  - Images: "image=1" (or "image=1:chunk=N")
  - Audio/video: "t=START-END" if available; else "transcript:chunk=N"
- Embeddings:
  - Use OpenAI embeddings API with model default "text-embedding-3-small".
  - Store embeddings as JSON array strings in embedding_json.

RAG FLOW
1) For a given asset:
- Embed user question
- Load all chunk embeddings for that asset (small dataset)
- Compute cosine similarity in Node
- Pick topK chunks (default 5)
2) Build evidence block:
- For each retrieved chunk: include [n], source_ref, and chunk text.
3) Ask OpenAI Responses API:
- Must answer using ONLY evidence
- Must cite [1], [2], etc.
- If insufficient, say “Not found in the provided file.”

MCP-STYLE TOOL ACTION (ONE “DO” FEATURE)
Create endpoint: POST /api/actions/extract-obligations
Input: { assetId }
Behavior:
- Retrieve evidence set (top chunks or all chunks if small)
- Call OpenAI Responses API with strict JSON output requirement:
  Return JSON schema:
  {
    "obligations": [
      {
        "who": "string",
        "must_do": "string",
        "when": "string|null",
        "source": "[n]"
      }
    ]
  }
- Must include citations via source field referencing evidence chunk numbers (e.g. "[2]").
- If none, return {"obligations": []}.

API ENDPOINTS
1) POST /api/upload
- Accept any file.
- Enforce max file size (25MB).
- Save asset row with status PROCESSING then READY/UNSUPPORTED/ERROR.
- Ingest according to mime/type detection.
- Return: { assetId, filename, mime, status, message }

2) POST /api/chat
Input: { assetId, question, topK }
- Do RAG retrieval and Responses call.
Return: {
  answer: string,
  citations: [{ n, source_ref, score }],
  evidencePreview: [{ n, source_ref, snippet }]
}

3) POST /api/actions/extract-obligations
Return: { obligations: [...] }
