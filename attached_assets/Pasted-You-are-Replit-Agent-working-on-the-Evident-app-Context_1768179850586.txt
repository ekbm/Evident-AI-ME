Implementation tasks (do all)

A) Python PDF Extraction Service
1) Add dependencies
- Create `services/pdf_extractor/requirements.txt`:
  fastapi
  uvicorn
  python-multipart
  pymupdf

2) Create FastAPI app
- `services/pdf_extractor/app.py` with:
  - POST `/health` -> { ok: true }
  - POST `/extract`:
     - inputs:
       - file: UploadFile (PDF)
       - docType: str (invoice|po|grn)
       - optional vendorHint, currencyHint
     - processing:
       - open PDF via PyMuPDF
       - extract text blocks with coordinates:
         page.get_text("dict") -> blocks/lines/spans with bbox
       - build `rawText` as concatenated spans by page
       - scanned detection:
         - total_text_chars = len(rawText.strip())
         - if total_text_chars < 200 OR (pages>0 and avg chars/page < 100) => isScannedLikely=true
       - field extraction heuristics:
         Implement simple regex + proximity search on `rawText`:
         - invoice number: patterns like (invoice|inv)\s*(no|number|#)[:\s]*([A-Z0-9\-\/]+)
         - PO reference: (po|purchase order)\s*(no|#)[:\s]*([A-Z0-9\-\/]+)
         - dates: (date|invoice date|due date) ... parse as best-effort (keep string if parse fails)
         - totals: find last occurrence of (total|amount due) and parse currency number
         - tax: (gst|vat|tax) parse amount
         - currency: detect AUD/USD/etc from symbols or hints
       - line items (MVP):
         - if a table is easily detectable: attempt a very simple row split:
            look for repeated lines containing qty+unit price+line total numbers
            otherwise return empty lineItems and let user add/edit in UI
       - anchors:
         - for each extracted field, attempt to locate the text in spans and record the first bbox match.
         - if not found, omit anchor for that field.
       - OCR hook (stub):
         - if isScannedLikely == true: return meta.isScannedLikely true and include message: "OCR_REQUIRED"
           DO NOT implement OCR fully unless trivial; just provide stub to integrate later.
     - return JSON as described.
