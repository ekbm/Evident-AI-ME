You are building “Citations & Sources” for Evident (evident-ai.net). The goal: every answer can show clear, trustworthy citations with consistent formatting and guardrails.

STACK (existing):
- Frontend: React 18 + TypeScript + Vite + Tailwind + shadcn/ui + Wouter + TanStack Query
- Backend: Node.js/Express + TypeScript
- DB: PostgreSQL + Drizzle ORM
- Storage: files already ingested into your system (PDF/DOCX/HTML/etc.)
- Optional external enrichment/web sources may exist in the product.

FEATURE: CITATIONS THAT FOLLOW GUIDELINES
Implement a citations pipeline + UI that ensures:
1) Every factual claim in an answer should be supported by at least one citation when available.
2) Citations must be anchored to the exact text evidence (quote/snippet) and location (page/section/line/timestamp).
3) Citations must clearly indicate SOURCE TYPE:
   - “Uploaded document”
   - “External web”
   - “User note / manual”
4) Citations must be deduplicated, stable, and clickable.
5) The UI must not be cluttered: footnote-style markers + expandable “Sources” panel.

STANDARD CITATION GUIDELINES (ENFORCE IN CODE + PROMPTING)
- Prefer primary sources (the user’s uploaded docs) over external sources.
- If multiple sources support a claim, show up to 2 citations per claim to reduce noise.
- Each citation must include:
  - title (file name or webpage title)
  - source type badge
  - author/publisher if known (for web)
  - date (doc created date if available; web publish date if available; otherwise retrieval date)
  - location pointer:
    - PDFs: page number + (optional) bounding box highlight coords if you have them
    - DOCX: section heading + paragraph index
    - HTML: heading + snippet offset
    - Audio/Video: timestamp range
  - evidence snippet (max 240 chars)
- Do not fabricate citations. If evidence is not found, mark “No supporting source found” and explain.
- If answer combines reasoning + sourced facts:
  - Tag sentences as [SOURCED] or [REASONED] internally
  - Only [SOURCED] sentences get citation markers.
- If user asks “show me where,” always open the source panel and jump to the cited location.

DATABASE CHANGES (Drizzle)
Add tables:
- citations:
  - id (uuid)
  - workspaceId
  - messageId (assistant message)
  - sourceId (points to an ingested file or external URL record)
  - sourceType enum: 'UPLOAD' | 'WEB' | 'MANUAL'
  - title
  - url (nullable)
  - fileId (nullable)
  - publisher (nullable)
  - author (nullable)
  - publishedAt (nullable)
  - retrievedAt (nullable)
  - locator JSON (required):
    - { type: 'pdf', page: number, bbox?: [x,y,w,h] }
    - { type: 'docx', heading?: string, paragraphIndex?: number }
    - { type: 'html', heading?: string, startOffset?: number }
    - { type: 'media', startSec: number, endSec: number }
  - snippet (string, required, <= 240 chars)
  - createdAt
- message_claims:
  - id (uuid)
  - messageId
  - claimText (string)
  - claimType enum: 'SOURCED' | 'REASONED' | 'UNSUPPORTED'
  - citationIds uuid[] (0..2 typical)
  - createdAt

SERVER LOGIC
- When generating an answer, the LLM must output structured JSON:
  {
    "answer_markdown": "... with inline markers like [^1] ...",
    "citations": [
      {
        "temp_id": "c1",
        "sourceType": "UPLOAD",
        "title": "Contract_SOW.pdf",
        "fileId": "...",
        "locator": { "type":"pdf", "page": 12 },
        "snippet": "…",
        "publisher": null,
        "author": null,
        "publishedAt": null
      }
    ],
    "claims":[
      {
        "text":"The SLA is 99.9% uptime.",
        "type":"SOURCED",
        "citation_temp_ids":["c1"]
      }
    ]
  }

- Validate server-side:
  - snippet exists and is <= 240 chars
  - locator exists
  - sourceType is correct
  - no more than N citations overall (default 20)
  - dedupe identical citations across same message
- Persist citations + claims in DB.
- If external sources enabled:
  - store retrievedAt and url
  - store page title and publisher if available
