(B) Browse / Quick actions (no question required)

  - Show Legal help topics only if Legal pack enabled.

========================================================
C) SERVER: ASK ROUTER (NO FREE-FORM HALLUCINATION)
========================================================
Implement:
- POST `/api/help/ask`
  body: { query: string }
  behavior:
   1) Fetch the same filtered topics as above.
   2) Compute a relevance score per topic using deterministic matching first:
      - tokenize query (lowercase, split on non-alphanum)
      - score for tag matches, title matches, area matches
      - add small boost if exact phrase match
   3) OPTIONAL: If you have an LLM already wired, you may use it ONLY to choose among top N topics.
      - The LLM must receive the candidate topic titles + descriptions + routes only.
      - The LLM must return ONLY topicId(s), no prose.
      - If LLM fails, fall back to deterministic scoring.
   4) Build the final response from the selected topic content (NOT LLM-generated instructions):
      - answer.title = topic.title
      - answer.body = topic.description
      - answer.steps = topic.steps
      - answer.routes = topic.routes
   5) Return HelpAskResult with top 3 matches.

Important: The instructions shown to the user must come from the topic object, not from the model.
