GOAL
From a transcript document (already uploaded / extracted), students can generate exam-ready study materials and have them saved inside Evident:
1) Exam Focus
2) Study Summary
3) Practice Questions
4) Flashcards
5) Cheat Sheet (one-page bullets)

UX CONSTRAINTS (VERY IMPORTANT)
- Do NOT add a new toolbar or lots of buttons.
- Do NOT clutter the UI.
- Add rotating placeholder hints in the Ask input (only when empty).
- Add subtle post-answer suggestions (small clickable text chips) under answers.
- Replace any “Export” wording with “Save”. Use Save/Saved only.

TECH STACK
- Backend: Node.js (Express-style routes likely)
- Frontend: React
- Use existing auth + workspace/document permission patterns.
- Use existing LLM provider/client already used for Q&A.

Table: study_materials

- user_id

- source_document_id (documentId for transcript)

- title (string)
- content (JSON preferred, otherwise TEXT)
- created_at

A) POST /api/study/generate

- Verify auth (current user)
- Verify permission to access workspace + document

- Choose prompt template based on type (see Prompt Templates below)

C) GET /api/study/:id
