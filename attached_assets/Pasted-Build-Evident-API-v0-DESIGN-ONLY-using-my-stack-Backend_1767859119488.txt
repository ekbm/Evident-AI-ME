========
LOCK 5 FUTURE-PROOF API SURFACES (DESIGN ONLY)
========

A) Ingestion API
1) POST /api/v0/ingest/document
Request body (Zod):
{ orgId, sourceId?, filename, mimeType, contentBase64? OR contentText? OR fileUrl? }
Rules:
- Must accept at least one of contentBase64/contentText/fileUrl
Behavior:
- Create Document row
- Create mock chunks (e.g., split contentText into paragraphs; if base64 or fileUrl, generate placeholder chunks)
- Write AuditLog event "INGESTED_DOCUMENT"
Response:
- Document

2) POST /api/v0/ingest/batch
Request:
{ orgId, sourceId?, items:[{ filename, mimeType, contentBase64? contentText? fileUrl? }] }
Behavior:
- Create batchId
- Insert documents + chunks
- AuditLog event per document
Response:
{ batchId, acceptedCount, documents:[Document] }

B) Readiness & Validation API
1) GET /api/v0/readiness/:docId
Behavior:
- If readiness exists, return it
- Else compute deterministically (no AI calls) based on:
  mimeType, sizeBytes, whether chunks exist, average tokenCount, presence of “scanned” keyword, etc.
- Example issues:
  "No extractable text", "Low structure", "Too large", "Image-based PDF suspected"
- Save ReadinessScore + AuditLog "COMPUTED_READINESS"
Response:
- ReadinessScore

2) GET /api/v0/orgs/:orgId/readiness-summary
Response:
{ orgId, docsScanned, avgScore, gradeBreakdown:{A,B,C,D,F}, topIssues:[{issue, count}] }

C) Extraction API
1) GET /api/v0/extract/:docId/entities
Response: Extraction with output:
{ entities:[{ type, value, confidence, evidenceChunkIds[] }] }

2) GET /api/v0/extract/:docId/tables
Response output:
{ tables:[{ title?, rows:[string[]] }] }

3) GET /api/v0/extract/:docId/relationships
Response output:
{ relationships:[{ from, to, type, confidence, evidenceChunkIds[] }] }

Behavior for all extraction endpoints:
- If extraction exists, return it
- Else generate mock-but-realistic output from chunk text (simple regex heuristics)
- Store Extraction row + AuditLog "RAN_EXTRACTION"

D) Knowledge API
1) GET /api/v0/knowledge/search?orgId=...&q=...&topK=...&filters=...
Response:
{ orgId, q, results:[{ docId, chunkId, score, snippet }] }
Behavior:
- Use a lightweight scoring heuristic (keyword overlap) over chunks in the org

2) GET /api/v0/knowledge/graph?orgId=...
Response: KnowledgeGraph
Behavior:
- Build nodes/edges from entity + relationship extractions
- Save KnowledgeGraph + AuditLog "GENERATED_GRAPH"

E) Audit & Trust API
1) GET /api/v0/audit/lineage?orgId=...&docId=...
Response:
{ orgId, docId?, lineage:[{ docId, sourceId, ingestedAt, readinessComputedAt?, extractionsRun:[kinds], lastUpdatedAt }] }

2) GET /api/v0/audit/last-updated?orgId=...
Response:
{ orgId, lastIngestAt, lastReadinessAt, lastExtractionAt, lastGraphAt }
