PHASE 4 — SMART INTENT LAYER (AUTO PERSONALIZATION)
Implement a lightweight “Smart Intent Layer” in the backend that detects what the user likely needs based on:
- document type (transcript vs contract/policy/report)
- question keywords and phrasing
- optional recent actions (what templates they clicked)

This does NOT require ML. Use heuristics scoring + confidence.
Create: server/intent/intentDetector.js and server/intent/suggestionEngine.js.

Intent outputs:
- intent: "student" | "professional" | "general"
- confidence: 0..1
- recommendedTemplates: array of template types (from the 5 study types for transcripts, and allow future expansion)

Heuristic rules (v1):
- If document looks like transcript (keywords: “lecture”, timestamps, speaker labels, or metadata indicates transcript):
  + student score
- If question includes: “exam”, “revise”, “flashcards”, “practice questions”, “cheat sheet”, “explain simply”:
  + student score
- If document looks like legal/compliance (keywords: “agreement”, “clause”, “liability”, “policy”, “audit”):
  + professional score
- If question includes: “risk”, “obligation”, “compliance”, “summarize for management”, “action items”:
  + professional score

Store minimal preference safely (optional, but recommended):
- If user repeatedly uses student templates, persist a small user_preference record:
  { user_id, preferred_intent, updated_at }
But do NOT require onboarding selection.

Add endpoint:
GET /api/intent/suggestions?workspaceId=&documentId=
Returns:
{
  "intent":"student",
  "confidence":0.82,
  "placeholderHints":[...],
  "postAnswerChips":[...],
  "recommendedTemplates":[...]
}

Frontend should call this to render context-aware placeholder hints and chips.
If endpoint is not available/fails, fallback to default generic hints.
