You are working in an existing app called “Evident”.
Evident already supports: document upload, AI readiness scan, issues list, and “Make AI-Ready” preparation pipeline (or stubs).

Goal:
Implement “Pilot Mode v1” so organisations can safely test Evident with a small scope and real documents.
Pilot Mode must be available for testing now, without building full enterprise governance.

Tech Stack (existing):
Frontend: React 18 + TypeScript + Vite + Tailwind + shadcn/ui + Wouter + TanStack Query
Backend: Node.js + Express + TypeScript
DB: PostgreSQL (Drizzle)

IMPORTANT PRINCIPLE:
Pilot Mode is NOT a separate product. It is a mode that applies guardrails + defaults to the same engine.
Same upload/scan/prep/extract — but stricter gating, limited scope, and clearer UI.

DO NOT implement StoreKit purchases.
DO implement mode switching + pilot screens + pilot gating rules + minimal “request assessment/expansion” lead capture.

2) PILOT SCOPE (limited by design)
--------------------------------------------------------------------

Pilot Mode requires a “pilot scope” record.

Pilot Scope v1 (minimal):
- pilotId
- name (default: “Organisation Pilot”)
- createdAt
- scopeType: 'UPLOADS' (v1 only; future: NAS/SMB/Drive)
- limits:
  - maxDocuments = 200 (config)
  - maxTotalSizeMB = 500 (config)
- status: ACTIVE

DB (Drizzle):
Create table: pilots
- id (uuid)
- name (text)
- scopeType enum('UPLOADS') // expand later
- maxDocuments int
- maxTotalSizeMB int
- createdAt timestamp
- status enum('ACTIVE','PAUSED') default 'ACTIVE'

Create table: pilot_documents
- pilotId
- documentId
- addedAt timestamp

Rules:
- In PILOT mode, uploads must attach documents to the active pilot.
- Enforce pilot limits on upload:
  - if exceeds maxDocuments or size, return 403 with a helpful error and “Request expansion” CTA.

Backend endpoints:
- POST /api/pilot/init -> creates (or returns) active pilot for current user/org
- GET /api/pilot -> returns active pilot + counts (docs, size)
- POST /api/pilot/add-documents -> attach uploaded docs to pilot
- GET /api/pilot/documents -> list docs in pilot with readiness + owner fields
- POST /api/pilot/request-expansion -> creates a request record (lead capture)

Note: If authentication/org does not exist yet, treat “pilot” as per-user local account, still using DB.

--------------------------------------------------------------------
3) PILOT GUARDRAILS (core value)
--------------------------------------------------------------------

Pilot Mode must enforce these rules:

A) Mandatory readiness scan gate:
- In PILOT mode, a document cannot be used for extraction/RAG until a readiness scan exists.
- If user attempts extraction/chat and no scan exists:
  return 409 { code:'SCAN_REQUIRED', message:'Run AI Readiness Scan before using this document in Pilot Mode.' }

B) Stricter readiness gating:
- Allow extraction only if:
  - latestScan.status === 'READY'
  OR user performs explicit override WITH reason (v1 reason required text input)
- Overrides are recorded in DB:
  pilot_overrides: id, pilotId, documentId, reason, createdAt

C) No “org-wide” knowledge in v1:
- Add scope parameter to extraction:
  scope: 'PRIVATE' | 'PILOT_SHARED'
- In PILOT mode, only allow 'PILOT_SHARED' within the pilot; do not implement ORG-wide.
- If user tries 'ORG_SHARED' (if exists in code), block with:
  403 { code:'PILOT_SCOPE_ONLY', message:'Pilot Mode limits usage to pilot scope only.' }

D) Owner accountability prompt (lightweight):
- Documents default to ownerDisplayName = 'Evident Intake (Unassigned)'
- In PILOT mode, when a user clicks “Use in Pilot” or “Share in Pilot”:
  - If owner is still Intake, show modal:
    “Assign an Owner for accountability before using this in the pilot.”
  - Allow selecting owner from owners list (at minimum include Intake + “Add custom owner name” v1).
  - Save assignedOwner fields (do not fabricate sourceAuthor).

E) Always-visible isolation copy:
- On pilot screens show a banner:
  “This pilot is isolated. Documents here are not used outside this pilot.”
