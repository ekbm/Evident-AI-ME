You are building “Evident Enterprise Agent + Org Admin Console” for a product called Evident.

PRIMARY GOAL
Deliver a secure, enterprise-friendly folder-ingestion agent framework (Windows + macOS) PLUS an Org Admin Console (React) where organizations can see/control all agents. ALSO: make MSI/PKG installers + IT Pack downloadable ONLY from inside the authenticated org admin area (role-gated), with signed/time-limited download links and audit logs.

IMPORTANT: This repo is an MVP for testing and self-serve pilots. Do NOT build real Windows Service or macOS LaunchDaemon binaries here. Instead:
- Build the backend + UI + DB + “agent simulator” that behaves like real agents.
- Provide placeholder MSI/PKG files and an IT Pack bundle (docs) that ships alongside them.
- Keep everything end-to-end working in Replit.

TECH STACK (MUST USE)
- Frontend: React 18 + TypeScript + Vite + Tailwind + shadcn/ui
- Routing: Wouter (preferred)
- Data fetching: TanStack React Query
- Backend: Node.js + Express + TypeScript
- DB: PostgreSQL + Drizzle ORM
- Auth: simple email login (can be stubbed) with org roles: OWNER | ADMIN | MEMBER
- Agent simulator: Node script that simulates multiple PCs.

PRODUCT PRINCIPLES (NON-NEGOTIABLE)
1) Explicit folder allowlist only (no whole disk scan).
2) Machine-wide deployment is the target (Windows Service + macOS LaunchDaemon) but in this MVP we simulate the agent core.
3) No inbound ports on the agent. Agents poll server for commands and send heartbeats (outbound only).
4) Org admins can see fleet status, issue commands, revoke devices.
5) Default upload mode is “extracted text + metadata only” (simulate extraction; no real PDF parsing required).
6) Audit everything important (downloads, token generation, device enrollment, revoke, policy changes).

DELIVERABLES (MUST BUILD)

A) ORG ADMIN CONSOLE (REACT)
Routes:
- /login
- /org/agents                (Admin/Owner: all devices; Member: only their devices)
- /org/agents/:id             (details)
- /org/policies               (Admin/Owner)
- /org/audit                  (Admin/Owner)
- /org/settings/agents         (Admin/Owner)  <-- installers + IT pack + token generation

A1) /org/agents page (Fleet)
- Summary cards: total agents, online, offline, paused, error, stalled, seats used/limit
- Table with filters: status, OS, owner, stale > 7 days, needs attention, outdated version
- Columns: Status badge, Device name, Owner, OS, Version, Last seen, Last sync, Folder count, Actions menu
- Actions (Admin/Owner only): Pause indexing, Resume indexing, Pause uploads, Resume uploads, Rescan, Revoke device
- Poll every 30 seconds with React Query

Status logic:
- online if now - last_seen_at < 2 * heartbeat_interval
- offline otherwise
- stalled if online AND queue_depth > 0 AND now - last_progress_at > STALL_THRESHOLD (e.g. 30 mins)
- error if last_state == "error" OR last_error_code exists
- paused if last_state == "paused"
- revoked if status_override == "revoked"

A2) /org/agents/:id page
- Device info card: OS, version, install_mode, owner, last seen, last sync, last progress, policy version applied
- Folder allowlist list (path can be masked)
- Timeline of last 50 device events
- Commands history and results
- Actions panel (Admin/Owner)

A3) /org/policies page (Admin/Owner)
Editable policy UI + JSON view. Policy fields:
- allowed_file_types []
- blocked_path_patterns []
- max_file_size_mb
- upload_mode: text_only | raw_allowed
- pii_mode: redact | block | allow
- citations_required: boolean
- minimum_sources: number
- do_not_guess_from_non_extractable: boolean
- heartbeat_interval_seconds
- command_poll_seconds
- freshness_max_age_days (optional)
- external_sources_allowed: boolean (optional; future)
Policies are versioned; devices report applied_policy_version.

A4) /org/audit page

A5) /org/settings/agents page (Admin/Owner only)

Sections:

  - Download Windows Agent (MSI)
  - Download macOS Agent (PKG)
  - Download IT & Security Pack (ZIP)
  - Download SHA256SUMS.txt

2) Enrollment

- Generate org enrollment token (long-lived, rotatable)

3) Recent security events

Members visiting this route should see a message:

   - pairing_code OR org_enrollment_token
  Enforces plan seat limits (org.plan_device_limit).
  Returns: device_id + agent_token

- GET /api/agent/commands/poll

  Saves command execution result
- POST /api/agent/events

- GET /api/org/downloads/:token
