D) WORKER
Create backend/src/queue/worker.ts
- Start a BullMQ Worker('ingest', handler)
- handler(job):
  - mark upload status='processing', step='start', startedAt=now, workerId, lastHeartbeatAt=now
  - start heartbeat interval: update lastHeartbeatAt every JOB_HEARTBEAT_INTERVAL_MS
  - run steps in order:
    1) OCR step (timeout 15min)
    2) chunk step (timeout 5min)
    3) embed step (timeout 10min)
    4) index step (timeout 5min)
  - update step before each stage
  - on success: status='completed', step='done', updatedAt=now
  - on failure: status='failed', errorMessage=error string, updatedAt=now, throw to BullMQ for retry
- Configure job options:
  - attempts: 3
  - backoff: exponential (e.g. 30s, 2m, 10m)
  - removeOnComplete: keep last 1000
  - removeOnFail: keep last 500

Use helper:
backend/src/utils/withTimeout.ts
- wraps a promise with timeout in ms, throws TimeoutError

Also add “cancellation check”:
- Before each step, read upload.status; if status='cancelled', throw CancelledError.
