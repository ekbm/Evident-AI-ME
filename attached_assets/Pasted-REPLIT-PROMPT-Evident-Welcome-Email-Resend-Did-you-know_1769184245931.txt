Part A — Send welcome email (Resend)

1) Data model: “send once” flag
Add one of these (choose based on your current user store):
Option 1 (Postgres):
- Add column on users table:
  welcome_email_sent_at TIMESTAMP NULL

Option 2 (Firestore user profiles):
- Add field:
  welcome_email_sent_at: serverTimestamp | null

Also store:
- created_at
- last_login_at

2) Trigger strategy (implement Option B now; optionally add Option A later)
Option B (recommended for simplicity): Server endpoint called once after first login
- Endpoint: POST /api/email/welcome
- Client calls it after login (or after user profile is created).
- Server verifies Firebase ID token (Authorization: Bearer <idToken>).
- Server checks welcome_email_sent_at:
  - If exists => return 204 (no-op)
  - If missing => send email via Resend, then set welcome_email_sent_at = now.

Important behaviors:
- Do NOT block login if email fails.
- Log a minimal error (no sensitive info).
- One retry max (optional).

3) Resend integration
- Install: resend npm package
- Env vars:
  RESEND_API_KEY
  EMAIL_FROM="Evident <no-reply@yourdomain.com>" (or your Resend verified sender)
  EMAIL_REPLY_TO="support@yourdomain.com" (optional)
- Use Resend API to send:
  - to: user.email (from verified domain)
  - subject: "Welcome to Evident"
  - html: rendered HTML template
  - text: plain text fallback
  - reply_to if configured
